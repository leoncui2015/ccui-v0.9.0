兼容的对象

CCUI是一个以兼容和简洁为核心的框架。CCUI把框架和UI分离，框架除了兼容主流的浏览器之外也向后兼容了IE6-IE9的较老版本的IE系列，UI在主流浏览器表现良好，注重简洁，我们把更多的DIY空间留给设计师。所以这里说的兼容的对象主要是针对IE6-IE9。

respond.src.js

前面我们使用 html5.js 来使IE6-9识别HTML5的标签。同样的我们只需要加载 respond.src.js 就可以使IE6-9支持CSS3的媒体查询。

更多点击 媒体查询 。

font-awesome-ie.min.css

国人制作的使font-awesome兼容IE7的样式，在CCUI框架下测试也兼容IE6。使用方法很简单，我们只需要加载 font-awesome-ie.min.css 就可以了。

更多点击 奥森图标 。

roundies.js

我们使用 roundies.js 来使IE6-9支持圆角属性。详细的文档请参考 DD_roundies 或JS文件内的示例部分。

例子

IE圆角边框

<div class="ie-round" style="padding:10px;border-radius:6px;border:1px solid #ccc;">IE圆角边框</div>

<script language="javascript" type="text/javascript">
	DD_roundies.addRule('.ie-round', '6px');
</script>

EvPng.js

我们使用 EvPng.js 来使IE6支持PNG透明属性。

例子 

<div class="ie-png"><img src="images/logo2.png" alt="" /></div>

<!--[if lte IE 6]>
<script src="js/EvPng.js"></script>
<script language="javascript" type="text/javascript">
	EvPNG.fix(".ie-png img");
	</script>       
<![endif]-->

使用方法也很简单，在引入 EvPng.js 后，设置需要透明的图像文件，如 .ie-png img ;如果图像文件在样式里引用的则设置成相对应的样式类或标签，如 .ie-png 。

CSS hack - 条件注释法

条件注释一般用来引用专门为IE准备的JS插件，如我们上述所讲的圆角，PNG透明等都是通过这个注释来实现的。 

<!--[if lte IE 6]>
...
<![endif]-->

CSS hack - 类内属性前缀法

即在类的属性前面添加各种浏览器版本的前缀。下面是一个典型的兼容性的例子，请注意从上到下的顺序，兼容都是从最新版本和主流的浏览器兼容属性开始写起，最后才是特殊浏览器的属性，现在我们最特殊的浏览器，当然是IE6了。 

color: #ccc;
color: #ccc\9; /* IE6-IE10 */
color: #ccc\9\0; /* IE9-IE10 */
color: #ccc\0; /* IE8-IE10 */
*color: #ccc; /* IE6-IE7 */
+color: #ccc; /* IE7 */
_color: #ccc; /* IE6 */

对于一些新的CSS属性，比如 border-radius 圆角，主流浏览器为了兼容他们较早版本的私有属性也有他们的专有hack。与IE的兼容不同的是，他们的顺序可随意排列。 

-webkit-border-radius: 24px; /* Chrome and Safari */
-moz-border-radius: 24px; /* Firefox */
-ms-border-radius: 24px; /* IE */
-o-border-radius: 24px; /* Opera */
border-radius: 24px;

CSS hack - 选择器前缀法

选择器前缀法是目前较新的方法，一般前面2种已经能解决80%的兼容问题，如果可以用上面的方法解决，一般不推荐首先用这种。 

@media screen and (-webkit-min-device-pixel-ratio:0) { 
.class-name{
	color: #ccc; /* Chrome and Safari */
}
}
@-moz-document url-prefix() { 
.class-name{
	color: #ccc; /* Firefox */
}
}
@media all and (min-width:0) { 
.class-name{
	color: #ccc; /* Opera */
}
}

CSS hack - IE选择器前缀法

如果碰到诸如IE8专属的hack等极限的问题的时候，就可以用这个IE选择器前缀法了，前提是让IE较老版本先支持媒体查询。 

/* IE 6/7 */
@media screen\9 {
 .class-name{
 	color: #ccc;
 }		
}
 
/* IE 6/7/8 */
@media \0screen\,screen\9 {
 .class-name{
 	color: #ccc;
 }		
}
 
/* IE 8 */
@media \0screen {
 .class-name{
 	color: #ccc;
 }	
}
 
/* IE 8/9/10 & Opera */
@media screen\0 {
 .class-name{
 	color: #ccc;
 }		
}
 
/* IE 9/10, Firefox 3.5+, Opera */
@media screen and (min-resolution: +72dpi) {
 .class-name{
 	color: #ccc;
 }		
}
 
/* IE 9/10 */
@media screen and (min-width:0\0) {
 .class-name{
 	color: #ccc;
 }		
}
 
/* IE 10+ */
@media screen and (-ms-high-contrast: active), (-ms-high-contrast: none) {
 .class-name{
 	color: #ccc;
 }		
}
 
/* Everything but IE 6/7/8 */
@media screen and (min-width: 400px) {
 .class-name{
 	color: #ccc;
 }		
}

兼容的技巧

1. IE6-10系列大部分的浮动 div 问题，可以通过设置层的 width height 还有背景色（调试用）解决。
2. 如果项目时间紧，可以只在首页上做兼容，其他的页面兼容只做布局方面，忽略掉细节。
3. 能用其他方法避免的话，尽量少写hack。
4. 重要的事情说三遍：要耐心，要耐心，要耐心。
